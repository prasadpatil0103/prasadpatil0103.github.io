NYC Taxi: Scalable Data Engineering Pipeline

Syracuse University | IST 782 | Data Engineering Track | Feb 2026 – Mar 2026

Project Overview

Architected an end-to-end production-grade ELT pipeline processing over 5.7 million New York City taxi trip records. This project simulates a real-world data platform using PySpark, Snowflake, and dbt — implementing the Medallion Architecture to organize data into Bronze, Silver, and Gold layers. Curated Snowflake tables are exposed through FastAPI services enabling real-time analytics dashboards.

The Technical Architecture (Medallion Architecture)

  • 🥉 Bronze Layer (Raw Ingestion): Ingested raw NYC taxi trip records into Snowflake without transformation, preserving source data for auditability and reprocessing.
  • 🥈 Silver Layer (Validated & Cleansed): Used PySpark for distributed data processing across 5.7M+ records. Applied validation rules, removed invalid entries, standardized schemas, and resolved data quality issues.
  • 🥇 Gold Layer (Analytics & ML-Ready): Implemented dbt transformation models producing curated datasets optimized for analytics and machine learning consumption — aggregated views for taxi demand by zone, fare distribution, distance analysis, and time-based pickup trends.
  • FastAPI Services: Exposed curated Snowflake warehouse tables through FastAPI enabling real-time analytics dashboards and downstream consumption.

Key Technical Achievements

  • Scale: Processed 5.7M+ records through a fully automated ELT pipeline.
  • Distributed Processing: PySpark parallel processing enabling efficient large-volume data transformation impractical with single-node tools.
  • Data Quality & Governance: Systematic Silver-layer validation ensuring data reliability across the pipeline — critical for production analytics environments.
  • dbt Best Practices: Modular SQL models, version control, and automated testing ensuring maintainable, trustworthy analytics outputs.

What This Project Demonstrates

This project reflects the core skills of the Data Engineering track — building systems that collect, process, and store data reliably before any analysis or ML is performed. It demonstrates that great data science depends on great data infrastructure.
